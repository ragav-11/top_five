# CI/CD Experience

## Ragav Rajan

During a software engineering internship at Qualcomm, I worked on Retrieval-Augmented Generation (RAG) projects where CI/CD pipelines were a core part of the development workflow. This included:

- Setting up and maintaining GitHub Actions workflows to automatically run linting, unit tests, and integration tests on every pull request
- Using automated pipelines to build and validate RAG pipeline components (document ingestion, embedding generation, retrieval evaluation) before merging
- Deploying model updates and backend services through CD pipelines to internal staging environments, reducing manual deployment steps

This experience gave me hands-on familiarity with writing workflow YAML files, debugging pipeline failures, and structuring projects so automated testing catches regressions early.
